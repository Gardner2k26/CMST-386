CSS Preprocessing

Using Sass to Extend CSS

Sass adds reusable programming features to CSS and makes larger stylesheets easier to organize, maintain, and update. The Sass source code is compiled into standard CSS before it is used by the web browser.

Implementation Methods

Client-Side and Server-Side Preprocessing

Client-Side Preprocessing

Client-side preprocessing occurs in the visitor's web browser. The page loads the preprocessing source file and uses a script to convert the source code into CSS while the website is loading. This method can be useful during testing because changes can be viewed quickly. However, it may increase loading time and depends on the visitor's browser and JavaScript. For those reasons, client-side preprocessing is not usually recommended for a finished website.

Server-Side or Development-Side Preprocessing

Server-side or development-side preprocessing converts the Sass file into a standard CSS file before the visitor opens the website. A command-line compiler, code editor extension, or development application performs the conversion. The browser then downloads only the finished CSS file. This method is preferred for production websites because the browser does not have to compile the Sass code, which creates a faster and more reliable experience.

Sass Features

Mechanisms Used in Sass

Sass provides several mechanisms that reduce repeated code and make stylesheets easier to update.

Variables

Variables store reusable values such as colors, fonts, spacing, widths, and responsive breakpoints. For example, the variable $bright-purple stores the main purple color used throughout this project. If that value is changed, every rule using the variable is updated when the Sass file is compiled.

Mixins

Mixins store groups of reusable style declarations. This project uses mixins for buttons, cards, and box shadows. A mixin can be added to multiple selectors with the @include command. This reduces repeated code and helps several elements maintain the same appearance.

Functions

Sass functions calculate or modify values. This project uses color functions such as lighten() and darken() to create lighter or darker versions of the main purple colors. Functions make it easier to create related colors without manually choosing every color value.

Operators

Operators perform mathematical calculations using addition, subtraction, multiplication, or division. In this project, the standard spacing variable can be multiplied to create a larger spacing value. Operators help maintain consistent measurements throughout a stylesheet.

Nested Selectors

Nested selectors place related rules inside a parent selector. For example, the links, list items, and hover states for the navigation menu are written inside the main navigation rule. This structure reflects the organization of the HTML and keeps related styles together.

Variables in Nested Rules

Sass variables can be used inside nested selectors and component rules. For example, navigation links inside the main navigation selector use the same color variables defined at the beginning of the stylesheet. This allows nested elements to share a consistent color palette while keeping their rules organized within the correct parent selector.

Project Implementation

How Sass Was Used in This Website

This website uses a Sass source file named styles.scss. The file contains variables for the color palette, fonts, spacing, maximum content width, and responsive breakpoints. Mixins are used for reusable card styles, buttons, and shadows. Sass color functions create lighter and darker versions of the purple colors. An operator calculates a larger spacing value from the standard spacing variable. Nested selectors organize navigation links, cards, buttons, breadcrumbs, and responsive hamburger menu styles. The source file is compiled into css/styles.css, which is the stylesheet loaded by both Project 3 HTML pages.
